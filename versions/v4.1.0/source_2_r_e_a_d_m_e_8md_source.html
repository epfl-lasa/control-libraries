3 ## `state_representation`
4 
5 This library provides a set of classes to represent **states** in **Cartesian** or **joint** spaces.
6 The classes define and combine variables such as position, velocity, acceleration and force into
7 a consistent internal representation used across the other libraries.
8 
9 Source: [state_representation](./state_representation)
10 
11 Dependencies: [Eigen3](https://eigen.tuxfamily.org/index.php?title=Main_Page)

15 ## `dynamical_systems`
16 
17 This library provides a collection of classes that behave as differential equations to calculate
18 a derivative of a state variable. For example, a position state input might
19 yield a desired velocity output. This can be used to generate time-invariant trajectories.
20 
21 Source: [dynamical_systems](./dynamical_systems)
22 
23 Dependencies: `state_representation`

27 ## `robot_model`
28 
29 This library allows the creation of a robot model from a URDF file and defines many helpful
30 rigid-body algorithms for solving kinematic and dynamic problems.
31 
32 It is a wrapper for [Pinocchio](https://github.com/stack-of-tasks/pinocchio)
33 that is compatible with the internal `state_representation` types.
34 
35 Source: [robot_model](./robot_model)
36 
37 Dependencies: `state_representation`, [Pinocchio](https://stack-of-tasks.github.io/pinocchio/download.html)

53 ## Installation
54 
55 ### Supported platforms
56 
57 These libraries have been developed and tested on Linux Ubuntu 18.04 and 20.04.
58 They should also work on macOS and Windows, though the installation
59 steps may differ. At this time no guarantees are made for library support on
60 non-Linux systems.
61 
62 ### Installation with the install script
63 This project uses CMake to generate static library objects for each of the modules.
64 
65 To facilitate the installation process, an [install script](./install.sh) is provided. Users who are interested in
66 the manual installation steps and/or have already installed Pinocchio refer to the
67 [manual installation steps](#manual-installation-steps) in the next section.
68 
69 The install script takes care of all the installation steps, including the installation and configuration of Pinocchio.
70 It can be run with several optional arguments:
71 - `-y`, `--auto`: Any input prompts will be suppressed and install steps automatically approved.
72 - `-d [path]`, `--dir [path]`: If provided, the installation directory will be changed to `[path]`.
73 - `--no-controllers`: The controllers library will be excluded from the installation.
74 - `--no-dynamical-systems`: The dynamical systems library will be excluded from the installation.
75 - `--no-robot-model`: The robot model library, and therefore Pinocchio, will be excluded from the installation.
76 - `--build-tests`: The unittest targets will be included in the installation.
77 - `--clean`: Any previously installed header files from `/usr/local/include` and any shared library files from
78  `/usr/local/lib` will be deleted before the installation.
79 - `--cleandir [path]`: Any previously installed header files shared library files from `[path]` will be deleted before
80  the installation.
81 
82 ### Manual installation steps
83 
84 Eigen3 is required for building `state_representation` and all other libraries.
85 You can install it with:
86 ```shell script
87 apt-get install libeigen3-dev
88 ```
89 
90 Pinocchio is required for building the `robot_model` library. Installing this requires
91 some additional steps; see the [install script](./install.sh) for reference.
92 If the `robot_model` library is not needed, you can skip the installation of Pinocchio.
93 
94 Once the dependencies are installed, build and install the libraries by navigating
95 to the source folder and invoking `cmake` and `make` as shown below.
96 The library files are installed to `usr/local/lib`, and the library header files
97 are copied to `/usr/local/include`.
98 
99 ```shell script
100 cd control-libraries/source
101 mkdir build && cd build
102 cmake -DCMAKE_BUILD_TYPE=Release ..
103 make -j
104 make install
105 ```
106 
107 The CMake configuration flags `BUILD_CONTROLLERS`, `BUILD_DYNAMICAL_SYSTEMS` and `BUILD_ROBOT_MODEL`
108 determine which libraries are built, and are all defined as `ON` by default.
109 The building of the `state_representation` library cannot be disabled, as all other libraries depend on it.
110 
111 To selectively disable the build of a particular library, set the flag to `=OFF`.
112 For example, the following flags will prevent the `robot_model` library from being built,
113 which is useful if the Pinocchio dependency is not fulfilled on your system.
114 
115 ```shell script
116 cmake -DCMAKE_BUILD_TYPE=Release -DBUILD_ROBOT_MODEL=OFF ..
117 ```
118 
119 To also build the library tests, add the CMake flag `-DBUILD_TESTING=ON`.
120 This requires GTest to be installed on your system. You can then use `make test` to run all test targets.
121 
122 Alternatively, you can include the source code for each library as submodules in your own CMake project,
123 using the CMake directive `add_subdirectory(...)` to link it with your project.

125 ## Troubleshooting

127 This section lists common problems that might come up when using the `control-libraries` modules.

129 ### Boost container limit compile error in ROS
130 When using the `robot_model` module in ROS and trying to `catkin_make` the workspace, it might produce the following error:
131 ```bash
132 /opt/openrobots/include/pinocchio/container/boost-container-limits.hpp:29:7: error: #error "BOOST_MPL_LIMIT_LIST_SIZE value is lower than the value of PINOCCHIO_BOOST_MPL_LIMIT_CONTAINER_SIZE"
133 #23 2.389 29 | # error "BOOST_MPL_LIMIT_LIST_SIZE value is lower than the value of PINOCCHIO_BOOST_MPL_LIMIT_CONTAINER_SIZE"
134 ```
135 In order to avoid this error and successfully `catkin_make` the workspace, make sure that the `CMakeList.txt` of the ROS
136 package contains all the necessary directives, i.e. on top of
137 ```bash
138 
139 list(APPEND CMAKE_PREFIX_PATH /opt/openrobots)
140 find_package(Eigen3 REQUIRED)
141 find_package(pinocchio REQUIRED)
142 find_package(OsqpEigen REQUIRED)
143 find_package(osqp REQUIRED)
144 
145 find_library(state_representation REQUIRED)
146 find_library(dynamical_systems REQUIRED)
147 find_library(controllers REQUIRED)
148 find_library(robot_model REQUIRED)
149 ...
150 include_directories(
151  ${catkin_INCLUDE_DIRS}
152  ${Eigen3_INCLUDE_DIRS}
153  ${STATE_REPRESENTATION_INCLUDE_DIR}
154  ${DYNAMICAL_SYSTEMS_INCLUDE_DIR}
155  ${ROBOT_MODEL_INCLUDE_DIR}
156  ${CONTROLLERS_INCLUDE_DIR}
157  ${PINOCCHIO_INCLUDE_DIR}
158  ${OsqpEigen_INCLUDE_DIR}
159  /opt/openrobots/include
160 )
161 ```
162 it should also have
163 ```bash
164 find_package(Boost REQUIRED COMPONENTS system)
165 add_compile_definitions(BOOST_MPL_CFG_NO_PREPROCESSED_HEADERS)
166 add_compile_definitions(BOOST_MPL_LIMIT_LIST_SIZE=30)
167 ```
168 For a comprehensive example, please check the [`CMakeLists.txt` of the ROS demos](../demos/ros_examples/CMakeLists.txt).
