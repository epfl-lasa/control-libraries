18 ## Base DynamicalSystem
19 
20 The base class DynamicalSystem defines the main public interface pattern which the derived classes follow.
21 
22 It is templated for any state type, though the intention is to use the **State Representation** library
23 (for example, `CartesianState` or `JointState`).
24 
25 The main function is:
26 ```c++
27 template<class S>
28 S DynamicalSystem<S>::evaluate(const S& state) const
29 ```
30 where `S` is the state representation type.
31 The purpose of the evaluate function is to calculate a state derivative from a state.
32 The current implementations support calculating a velocity from a position in either joint space or Cartesian space,
33 though higher derivatives and other space types can be extended in a similar fashion.
34 
35 In the case of a `CartesianState`, only the position and orientation is evaluated, and the returned object
36 updates only the linear and angular velocity properties.
37 In the case of `JointState`, only the joint position is evaluated, and the returned object contains only the
38 joint velocity.
39 
40 ### Base frame
41 
42 The *DynamicalSystem* base class has a private `base_frame` property, which can be thought of as the DS origin.
43 The functions `get_base_frame()` and `set_base_frame(const S& state)` can be used to access or modify this base frame.
44 
45 The `DynamicalSystem<CartesianState>` can be constructed with a `state` to set the base frame, or with string frame name. In
46 the latter case the base frame is set as a null / identity frame with the specified name.
47 For example, `DynamicalSystem<CartesianState>("base")` will create a dynamical system with the null base frame "base",
48 expressed in its own frame "base".
49 
50 The `DynamicalSystem<JointState>` can only be constructed with a `state` to set the base frame. Whilst the term
51 *base frame* makes more sense for a `CartesianState` DS, it rather refers to a specific robot with corresponding name
52 and joint names in the case of a `JointState` DS.
53 
54 In most cases, the constructor for the base DynamicalSystem should not be used directly,
55 and rather the derived DS classes should construct the base accordingly.
56 
57 However, the `set_base_frame()` method remains useful in combination with derived classes.
58 
59 ### Reference frames
60 
61 The following section applies to derived DS classes using the `CartesianState` type. The [Linear DS](#linear)
62 will be used as an example.
63 
64 The `evaluate()` function will always return a twist expressed in the same reference frame as the input state,
65 provided that the input state is compatible with the DS.
66 
67 The input state must be expressed in one of two supported reference frames:
68 1. The reference frame of the input is the DS base frame
69 2. The reference frame of the input matches the reference frame of the DS base frame
70 
71 The following snippet illustrates the difference in these two options.
72 ```c++
73 // create a linear DS with attractor B in frame A
74 state_representation::CartesianState BinA = state_representation::CartesianState::Identity("B", "A");
75 dynamical_systems::Linear<state_representation::CartesianState> linearDS(BinA);
76 
77 linearDS.get_attractor().get_name(); // "B"
78 linearDS.get_attractor().get_reference_frame(); // "A"
79 linearDS.get_base_frame().get_name(); // "A"
80 linearDS.get_base_frame().get_reference_frame(); // "A"
81 
82 // evaluate a point C in frame A
83 state_representation::CartesianState CinA = state_representation::CartesianState::Random("C", "A");
84 auto twist0 = linearDS.evaluate(CinA); // valid, twist is expressed in frame A
85 
86 // set the base from of the DS to be A expressed in the world frame
87 state_representation::CartesianState AinWorld = state_representation::CartesianState::Identity("A", "world");
88 linearDS.set_base_frame(AinWorld);
89 
90 linearDS.get_attractor().get_name(); // "B"
91 linearDS.get_attractor().get_reference_frame(); // "A"
92 linearDS.get_base_frame().get_name(); // "A"
93 linearDS.get_base_frame().get_reference_frame(); // "world"
94 
95 // option 1: reference frame of the input is the same as the base frame
96 // -> reference frame of the input: "A"
97 // -> DS base frame: "A"
98 // -> reference frame of the output: "A"
99 auto twist1 = linearDS.evaluate(CinA);
100 
101 // option 2: reference frame of the input is the same as the base frame
102 // -> reference frame of the input: "world"
103 // -> DS base frame reference frame: "world"
104 // -> reference frame of the output: "world"
105 auto CinWorld = AinWorld * CinA;
106 auto twist2 = linearDS.evaluate(CinWorld);
107 
108 // as a note, you can mix and match the approach as necessary.
109 // the following is using option 1 with an additional external operation
110 // to yield a final result equivalent to option 2.
111 auto twist3 = AinWorld * linearDS.evaluate(CinA);
112 // twist2 === twist3
113 ```
114 
115 Note that the base frame can have its own velocity or other state properties, which
116 are automatically combined with the DS result with respect to the common reference frame.
117 
118 Setting the base frame of the DS has some benefits. In some cases, the state variable to be
119 evaluated is not directly expressed in the frame of the DS.
120 Similarly, the output twist may need to be expressed in a different reference frame.
121 
122 As a practical example, consider a case where the state of an end-effector is
123 reported in the reference frame of a robot, while a linear attractor is expressed
124 in some moving task frame.
125 The robot controller expects a twist expressed in the robot frame.
126 By updating the DS base frame with respect to the robot frame,
127 any pre-transformation of the end-effector state or post-transformation of the twist can be avoided.
128 
129 ```c++
130 state_representation::CartesianState EE("end_effector", "robot");
131 state_representation::CartesianState attractor = state_representation::CartesianState::Random("attractor", "task");
132 state_representation::CartesianState taskInRobot("task", "robot");
133 
134 dynamical_systems::Linear<state_representation::CartesianState> linearDS(attractor);
135 
136 // control loop
137 while (...) {
138  // update the EE state from robot feedback
139  EE.set_pose(...);
140 
141  // update the state of the task with respect to the robot (for example, from optitrack)
142  taskInRobot.set_pose(...);
143  taskInRobot.set_linear_velocity(...);
144  taskInRobot.set_angular_velocity(...);
145 
146  // now update the DS base frame
147  linearDS.set_base_frame(taskInRobot);
148 
149  // find the twist in the robot reference frame
150  // directly from the end-effector position in the robot reference frame
151  auto twist = linearDS.evaluate(EE);
152 
153  // send twist command to controller
154  update_controller(twist);
155 }
156 ```

158 ## Linear
159 
160 The **Linear** DS can be thought of as a point attractor, with a velocity that is linearly proportional
161 to the distance of the current state from the attractor.
162 It is currently implemented for the `CartesianState` and `JointState` types.
163 
164 ```c++
165 // empty construction results in empty base frame and attractor
166 dynamical_systems::Linear<S> emptyDS;
167 
168 // construction with an attractor and default value for the gain
169 state_representation::CartesianState cartesianAttractor = state_representation::CartesianState::Identity("A");
170 dynamical_systems::Linear<state_representation::CartesianState> linearDS1(cartesianAttractor);
171 
172 state_representation::JointState jointAttractor = state_representation::JointState::Zero(6);
173 dynamical_systems::Linear<state_representation::JointState> linearDS2(jointAttractor);
174 ```
175 
176 ### Configuring the Linear DS
177 
178 The Linear DS has the following core parameters:
179 - **attractor**; the `CartesianState` or `JointState` type object defining the attractor pose relative to the DS base frame
180 - **gain**; the proportional gain acting towards the attractor
181 
182 Each parameter has corresponding `set_` and `get_` functions.
183 
184 To change the strength of the attractor, a gain can be passed as the second argument during construction or
185 passed to the `set_gain()` member function. The gain defines the proportionality between
186 a distance unit and a velocity unit, and is internally stored as a square matrix with a size corresponding
187 to the degrees of freedom in the state representation. For example, the `CartesianState` has six degrees of freedom
188 (XYZ in linear and angular space), while the `JointState` would have as many degrees of freedom as joints.
189 The gain can be defined as a matrix directly, as a diagonal vector of the appropriate
190 length, or as a scalar (which sets the value along the diagonal elements of the matrix).
191 ```c++
192 // set a gain (scalar, vector or matrix during construction)
193 double gain = 10;
194 state_representation::CartesianState csA = state_representation::CartesianState::Identity("A");
195 dynamical_systems::Linear<state_representation::CartesianState> linear(csA, gain);
196 
197 // or set / update the gain for the created object
198 std::vector<double> gains = {1, 2, 3, 4, 5, 6};
199 linear.set_gain(gains);
200 
201 // update the attractor
202 state_representation::CartesianState csB = state_representation::CartesianState::Random("B");
203 linear.set_attractor(csB);
204 ```
205 
206 ### Evaluating the Linear DS
207 
208 To get the velocity from a state, simply call the `evaluate()` function.
209 
210 ```c++
211 state_representation::CartesianState csA = state_representation::CartesianState::Identity("A");
212 dynamical_systems::Linear<state_representation::CartesianState> linear(csA);
213 
214 state_representation::CartesianState csB = state_representation::CartesianState::Random("B");
215 // note: the return type of evaluate() is a CartesianState, but
216 // it can be directly assigned to a CartesianTwist because the =operator
217 // has been defined for that purpose
218 state_representation::CartesianTwist twist = linear.evaluate(csB);
219 ```
220 
221 The returned velocity will always be expressed in the same reference frame as the input state.

223 ## Circular
224 
225 The **Circular** DS is a limit cycle that rotates around a center point in an elliptical orbit,
226 converging to a desired radius on a plane.
227 
228 The direction of the rotation is positive around the local Z axis;
229 this appears as a counter-clockwise rotation when viewed from "above".
230 
231 This DS is defined only for the `CartesianState` type. In addition, it only acts in linear space,
232 determining a linear velocity for a given position. It does not produce any angular velocity.
233 
234 The Circular DS can be constructed with a `CartesianState` state and radius as an argument;
235 the state position defines the center of the limit cycle, while the state orientation defines the inclination
236 of the limit cycle plane. The radius has a default value of 1.
237 
238 The internal representation of the limit cycle is a `state_representation::Ellipsoid` type. When a state is used
239 in the constructor, the limit cycle has a constant radius.
240 If an elliptical limit cycle is desired, the DS can be constructed directly from an `Ellipsoid` type.
241 
242 ```c++
243 // empty construction results in empty base frame and limit cycle
244 dynamical_systems::Circular emptyDS;
245 emptyDS.set_limit_cycle(ellipse); // sets a null base frame according to the reference frame of the limit cycle
246 
247 // construct the circular DS limit cycle using a CartesianState center
248 state_representation::CartesianState center = state_representation::CartesianState::Identity("center");
249 
250 // default constructor (radius = 1)
251 dynamical_systems::Circular circularDS1(center);
252 
253 // constructor with radius
254 double radius = 2.0;
255 dynamical_systems::Circular circularDS2(center, radius);
256 
257 // construct the circular DS limit cycle directly using an Ellipsoid type
258 state_representation::Ellipsoid ellipse("limit_cycle");
259 ellipse.set_center_state(center);
260 ellipse.set_axis_lengths({radius, 2 * radius});
261 dynamical_systems::Circular ellipticalDS(ellipse);
262 ```
263 
264 ### Configuring the Circular DS
265 
266 The Circular DS has the following core parameters:
267 
268 - **limit_cycle**; the `Ellipsoid` object defining the limit cycle center, shape and inclination.
269 - **planar_gain**; the proportional gain acting in the local plane towards the limit cycle radius.
270 - **normal_gain**; the proportional gain acting towards the local plane.
271 - **circular_velocity**; the expected angular orbital velocity around the local origin.
272 Setting this value negative reverses the direction of rotation. [rad/s]
273 
274 Each parameter has corresponding `set_` and `get_` functions.
275 
276 The constructor takes additional optional arguments to define the gain and circular velocity.
277 The scalar value for the gain sets both the planar and normal gain of the DS.
278 
279 Note that the argument for the radius is only present when a `CartesianState` is provided,
280 and not when an `Ellipsoid` is provided, as the radii are already parameters of the `Ellipsoid`.
281 ```c++
282 // construct the circular DS with optional parameters (default values are shown)
283 double radius = 1.0;
284 double gain = 1.0;
285 double circular_velocity = M_PI / 2;
286 
287 // construction with a CartesianState center
288 dynamical_systems::Circular circularDS(center, radius, gain, circular_velocity);
289 
290 // construction with an Ellipsoid ellipse
291 dynamical_systems::Circular circularDS(ellipse, gain, circular_velocity);
292 ```
293 
294 In addition, the following helper are defined to configure the limit cycle behaviour.
295 
296 ```c++
297 .set_center(CartesianState("center")); // sets the center position and orientation of the limit cycle
298 .set_gain(1.0); // sets both planar and normal gain to the gain value
299 .set_rotation_angle(0.0); // sets a rotation offset around the local Z axis
300 .set_radius(1.0); // sets both elliptical axes to the given radius (forces circle)
301 .set_radiuses({1.0, 2.0}); // sets the length of each elliptical axis individually
302 ```

305 ## Ring
306 
307 The **Ring** DS is similar to the **Circular** DS but is more parameterizable.
308 In brief, the strength of the limit cycle can be configured with a width around the ring radius.
309 
310 The direction of the ring orbit is a positive rotation around the local Z axis;
311 this appears as a counter-clockwise rotation when viewed from "above".
312 A clockwise rotation can be achieved by rotating the ring 180 degrees about its local X or Y axis.
313 
314 It only supports the `CartesianState` type, and always acts in a circular ring.
315 
316 ```c++
317 // empty construction results in empty base frame and center state
318 dynamical_systems::Ring emptyDS;
319 
320 // construction with center state and default values for parameters
321 state_representation::CartesianState center = state_representation::CartesianState::Identity("center");
322 dynamical_systems::Ring ringDS(center);
323 ```
324 
325 ### Configuring the Ring DS
326 
327 The Ring DS has the following core parameters:
328 
329 - **center**; the ring center `CartesianPose` expressed in the DS base reference frame.
330 This sets both the origin center and the inclination of the ring plane.
331 - **rotation_offset**; the orientation offset of the orientational attractor in the ring frame.
332 - **radius**; the ring radius. [m]
333 - **width**; the distance from the radius where the velocity has tangential components around the ring.
334 Beyond this width, the velocity is always perpendicular towards the radius. [m]
335 - **speed**; the desired linear speed when travelling along the circle radius.
336 The limit cycle is only stable when the speed is positive. [m/s]
337 - **field_strength**; the scale factor applied to the ring speed outside of the radius + width zone.
338 - **normal_gain**; the scale factor for the speed normal to the ring plane.
339 - **angular_gain**; the scale factor for angular error restitution.
340 
341 Each parameter has corresponding `set_` and `get_` functions.
342 
343 The constructor takes additional optional arguments to define the ring DS parameters.
344 ```c++
345 // construct the ring DS with optional parameters (default values are shown)
346 double radius = 1.0;
347 double width = 0.5;
348 double speed = 1.0;
349 double field_strength = 1.0;
350 double normal_gain = 1.0;
351 double angular_gain = 1.0;
352 dynamical_systems::Ring ringDS(center, radius, width, speed, field_strength, normal_gain, angular_gain);
353 ```
