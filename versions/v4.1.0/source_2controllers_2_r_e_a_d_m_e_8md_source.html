1 # Controllers
2 
3 This library introduces a set of controllers to be used in robotic control-loop schemes. Each controller is templated
4 to accepted an input space from `state_representation` and is derived from the `Controller` base class, e.g.,
5 
6 ```cpp
7 Impedance<state_representation::JointState> joint_space_impedance;
8 Disspative<state_representation::CartesianState> task_space_dissipative;
9 ```
10 
11 The main functions of each controllers are the `compute_command` variations. They implement the main algorithms of the
12 controllers. Depending on the controller, they are used to define a command from some input parameters.
13 
14 ```cpp
15 using namespace state_representation;
16 
17 Impedance<JointState> joint_space_impedance;
18 JointVelocities desired_velocities;
19 JointVelocities real_robot_velocities;
20 
21 JointTorques command = joint_space_impedance.compute_command(desired_velocities,
22  real_robot_velocities)
23 ```
24 
25 Some `compute_command` variations allows to transform one space to the other using extra parameters.
26 
27 ```cpp
28 using namespace state_representation;
29 
30 Dissipative<CartesianState> task_space_dissipative;
31 CartesianTwist desired_twist;
32 CartesianTwist real_eef_twist;
33 Jacobian robot_jacobian;
34 
35 JointTorques command = task_space_dissipative.compute_command(desired_twist,
36  real_eef_twist,
37  jacobian)
38 ```
39 
40 The above formulation is equivalent to:
41 
42 ```cpp
43 using namespace state_representation;
44 
45 Dissipative<CartesianState> task_space_dissipative;
46 CartesianTwist desired_twist;
47 CartesianTwist real_eef_twist;
48 Jacobian robot_jacobian;
49 
50 CartesianWrench task_command = task_space_dissipative.compute_command(desired_twist,
51  real_eef_twist)
52 
53 JointTorques joint_command = robot_jacobian.transpose() * task_command;
54 ```

56 ## Creating a new controller
57 
58 To create a new controller, you need to create a class that derives from the `Controller` base class or any derived
59 controller such as `Impedance`. This class can be templated to accept different input spaces (e.g. `CartesianState`
60 or `JointState`) or specify the desired input space.
61 
62 ```cpp
63 using namespace state_representation;
64 
65 class MyController:
66 public Controller<CartesianState> {
67 private:
68  ...
69 public:
70  CartesianState compute_command(const CartesianState& desired_state,
71  const CartesianState& feedback_state);
72 };
73 ```
74 
75 The only thing to implement is then the desired variation of the `compute_command` functions.
76 
77 ```cpp
78 CartesianState MyController::compute_command(const CartesianState& desired_state,
79  const CartesianState& feedback_state) {
80  CartesianState command = ...;
81  return command;
82 }
83 ```
84 
85 The main goal of this library is to reuse as much as possible the functionalities between controllers. Therefore, if
86 your controller is a variation of an impedance controller that simply adds extra way of computing the damping or
87 stiffness matrix (e.g. `Dissipative`), it should extend from it and reuse the main function calls.
88 
89 ```cpp
90 using namespace state_representation;
91 
92 class MyImpedanceController:
93 public Impedance<CartesianState> {
94 private:
95  ...
96 public:
97  CartesianState compute_command(const CartesianState& desired_state,
98  const CartesianState& feedback_state);
99 };
100 
101 CartesianState MyImpedanceController::compute_command(const CartesianState& desired_state,
102  const CartesianState& feedback_state) {
103  // do extra stuff like compute the damping or stiffness matrix
104  ...
105  CartesianWrench command = this->Impedance<CartesianState>(desired_state, feedback_state);
106  // eventually do extra stuff before returning the command
107  ...
108  return command;
109 }
110 ```
111 
112 With this implementation, you don't need to specify anything specific for using the space changing functionalities, it will be derived from the inheritance chain.
113 
114 ```cpp
115 CartesianState MyImpedanceController::compute_command(const CartesianState& desired_state,
116  const CartesianState& feedback_state,
117  const Jacobian& jacobian) {
118  // simply call the compute_command from impedance
119  return this->Impedance<CartesianState>(desired_state, feedback_state, jacobian);
120 }
121 ```
