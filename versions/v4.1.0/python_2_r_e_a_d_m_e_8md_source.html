5 ## Installation
6 
7 You must first install `control-libraries` before you can install the Python bindings.
8 Refer to the installation instructions in the top-level [README](../README.md) for more information.
9 
10 Additionally, the installation of the bindings requires the following prerequisites:
11 - `python3` >= 3.0
12 - `pip3` >= 10.0.0
13 
14 The installation itself is then quite straightforward:
15 ```shell script
16 git clone https://github.com/epfl-lasa/control-libraries
17 
18 ## install control-libraries (skip this stage if already done)
19 bash control-libraries/source/install.sh
20 
21 ## install the bindings using the pip installer
22 pip3 install control-libraries/python
23 ```
24 
25 The example above installs the module to the default dist-packages location.
26 You can see more information about the installed module using `pip3 show control-libraries`.
27 
28 The process also works with Python virtual environments. For example, with `pipenv`:
29 ```shell script
30 ## pip3 install pipenv
31 
32 pipenv install control-libraries/python
33 ```
34 
35 Once installed, you can simply import the module with an optional short alias:
36 ```python
37 #!/usr/bin/env python
38 import state_representation as sr
39 
40 print(sr.__version__)
41 
42 A = sr.CartesianState.Random("A")
43 print(A)
44 ```

46 Or, directly import specific classes from the module.
47 ```python
48 #!/usr/bin/env python
49 from state_representation import JointState
50 
51 B = JointState.Random("B", 3)
52 ```

54 If the `clproto` C++ library is installed, the installation steps above will automatically install the `clproto`
55 Python module which can be used to encode and decode objects into bytes of serialized data.
56 ```python
57 #!/usr/bin/env python
58 from state_representation import JointState
59 import clproto
60 
61 B = JointState.Random("B", 3)
62 encoded_msg = clproto.encode(B, clproto.MessageType.JOINT_STATE_MESSAGE)
63 
64 decoded_object = clproto.decode(encoded_msg)
65 ```

97 ## About
98 
99 [PyBind11](https://PyBind11.readthedocs.io/en/stable/index.html) is used to generate
100 Python bindings for the classes and functions in control libraries.
101 
102 The generated package is named `control-libraries`, but contains specific modules for importing.
103 These are named the same as the standard modules of control libraries (e.g. `state_representation`).
104 
105 The contents of the [`source`](./source) directory define the bindings between
106 each Python module and the respective C++ library. The source files to bind each module are
107 contained within a subdirectory of the same name.
108 
109 The `setup.py` and `pyproject.toml` files are used to configure the build and installation
110 of the Python bindings. The `.toml` file allows `pip` to automatically fetch the
111 installation dependencies (namely `setuptools` and `pybind11`) in a temporary cache,
112 allowing the subsequent `setup.py` to be evaluated without needing a local installation of `pybind11`.
113 This feature requires a [`pip`](https://pypi.org/project/pip/) version 10.0 or newer.
114 
115 The [`test`](./test) directory contains some Python scripts that import and check the bindings
116 using the Python `unittest` framework. They are not currently comprehensive.
117 
118 ## Dockerfile
119 
120 A Dockerfile and run script are provided to test the installation of the bindings.
121 
122 The docker image installs the core control libraries and subsequently installs the python bindings.
123 
124 The [`run.sh`](./run.sh) script will build the docker image and launch an interactive container
125 with the test files in the [`test`](./test) directory copied to the local path.
126 
127 The run script tries to the clone the current local git branch when installing the control libraries
128 in the Dockerfile. If the local branch does not exist on the remote, or if you want to test the
129 python bindings against a difference control libraries source branch, you can supply a specific
130 branch as the first argument to the run script. For example, `./run.sh develop` to use the `develop` branch.
131 
132 You can run a single test with `python <test_name.py>`, or just enter a python shell with `python`.
133 Run all tests with `python -m unittest discover <path_to_test_dir>`, or just `python -m unittest` if
134 the [`test`](./test) directory in your current working directory.
