1 # Control Libraries Protocol
2 
3 This module serves to define the protocol for sending and receiving
4 messages containing control libraries data across any network.
5 
6 The basis of the module is the Google Protocol Buffer, [protobuf](https://developers.google.com/protocol-buffers).
7 
8 ## Structure
9 
10 The [protobuf](./protobuf) directory contains the raw .proto message definitions
11 as well as a set of bindings by language that are generated by
12 the protobuf compiler (protoc). These bindings are not intended
13 to be directly user-facing.
14 
15 The [clproto_cpp](./clproto_cpp) directory contains the actual user-facing library,
16 with the translation between control libraries data and proto
17 message formats hidden behind a neat header file. The API
18 allows compatible objects to be easily encoded into a serialized
19 binary string for transmitting across. Equivalently, well-formatted
20 string messages can be decoded back into the equivalent objects.
21 
22 ## Installation
23 
24 The C++ `clproto` library requires control libraries [`state_representation`](../source/state_representation/README.md)
25 and [Google Protobuf](https://github.com/protocolbuffers/protobuf/blob/master/src/README.md)
26 to be installed on your computer, which includes the compiler `protoc` and the runtime library `libprotobuf.so`.
27 
28 An [install script](./install.sh) is provided in this directory. Run `./install.sh -h` for more information.
29 
30 ### Automatic dependency installation
31 
32 The easiest way to install `clproto` is to use the included install script in automatic mode.
33 By supplying the `--auto` flag to this script, it will automatically and recursively install any dependencies.
34 If Protobuf is not yet installed, this step will take some time.
35 ```shell
36 git clone https://github.com/epfl-lasa/control-libraries.git
37 sudo control-libraries/clproto/install.sh --auto
38 ```
39 
40 ### Copying protobuf dependencies
41 
42 If you are using Docker, the Protobuf dependencies are already built in the [`development-dependencies`](ghcr.io/epfl-lasa/control-libraries/development-dependencies:latest) image.
43 Since building and installing Protobuf from source takes quite a long time, you can instead copy the final artefacts
44 from this image into your image using docker `COPY` functionality:
45 
46 ```Dockerfile
47 COPY --from=ghcr.io/epfl-lasa/control-libraries/development-dependencies:latest /usr/local/include/google /usr/local/include/google
48 COPY --from=ghcr.io/epfl-lasa/control-libraries/development-dependencies:latest /usr/local/lib/libproto* /usr/local/lib
49 COPY --from=ghcr.io/epfl-lasa/control-libraries/development-dependencies:latest /usr/local/bin/protoc /usr/local/bin
50 RUN ldconfig
51 ```
