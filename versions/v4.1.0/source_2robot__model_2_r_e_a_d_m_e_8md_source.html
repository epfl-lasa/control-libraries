1 # Robot Model
2 
3 The `robot_model` library is a wrapper for the [Pinocchio](https://github.com/stack-of-tasks/pinocchio) library, used
4 to compute dynamics model of a robot.
5 It is specifically designed to work seamlessly with the `state_representation` library, offering conversion functions
6 between the states and input/output needed for `pinocchio`.
7

13 ## Model initialization
14 
15 The library offers a single class `Model` that encapsulates most of the functionalities from `pinocchio`, as well
16 as extra functions.
17 A `Model` is defined by a robot name, and initialized from a `URDF` description file of the robot.
18 
19 ```cpp
20 robot_model::Model model("myrobot", "path/to/urdf/myrobot.urdf")
21 ```
22 
23 At creation, it uses the parser from `pinocchio` to read the `URDF` file and fill the needed parameters for computation.
24 
25 ## Robot kinematics
26 
27 Most common robotics functionalities are implemented such `forward_kinematics`, `inverse_kinematics`,
28 `forward_velocity, `inverse_velocity`, ...
29 
30 ```cpp
31 // forward kinematics: pose of end-effector frame from the robot joint positions
32 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
33 state_representation::CartesianPose pose = model.forward_kinematics(jp);
34 // inverse kinematics: joint positions from end-effector pose
35 state_representation::CartesianPose cp = state_representation::CartesianPose::Random("eef");
36 state_representation::JointPositions jp = model.inverse_kinematics(cp);
37 // forward velocity kinematics: twist of end-effector frame from the robot joint velocities and positions
38 state_representation::JointState js = state_representation::JointState::Random("myrobot", 7);
39 state_representation::CartesianTwist twist = model.forward_velocity(js);
40 // inverse velocity kinematics: joint velocities from end-effector twist and current state of the robot
41 state_representation::CartesianTwist ct = state_representation::CartesianTwist::Random("eef");
42 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
43 state_representation::JointVelocities jv = model.inverse_velocity(ct, jp);
44 ```
45 
46 All of those functions check for inconsistencies between the model and the inputs (incompatibility of joints, frame
47 non existent in the robot model, ...).
48 Thus, they can be called on other frames than the end-effector (which is the default value), or even vector of frames
49 for bulk operations.
50 
51 ```cpp
52 // pose of multiple robot frames from the robot joint positions
53 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
54 auto poses = model.forward_kinematics(jp, std::vector<std::string>{"joint2", "eef_link"});
55 ```
56 
57 The Jacobian of the robot can also be computed and stored in the `state_representation::Jacobian` wrapper:
58 
59 ```cpp
60 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
61 state_representation::Jacobian = model.compute_jacobian(jp);
62 ```
63 
64 It can also be computed on lower frames than the end-effector, returning then a subset of the matrix, where non used
65 joints are filled with 0 values.
66 
67 ```cpp
68 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
69 state_representation::Jacobian = model.compute_jacobian(jp, "joint3");
70 ```

72 ## Robot dynamics
73 
74 Dynamic modeling of the robot is available within `pinocchio` and allows the computation of the gravity, coriolis, and
75 inertia torques as well as their matrix counterpart.
76 
77 ```cpp
78 // inertia matrix from joint positions
79 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
80 Eigen::MatrixXd inertia = model.compute_inertia_matrix(jp);
81 // inertia torques, equivalent to inertia multiplied by joint accelerations. The joint positions part of the state
82 // is used to compute the inertia matrix
83 state_representation::JointState js = state_representation::JointState::Random("myrobot", 7);
84 state_representation::JointTorques inertia_t = model.compute_inertia_torques(js);
85 
86 // coriolis matrix from joint positions and velocities
87 state_representation::JointState js = state_representation::JointState::Random("myrobot", 7);
88 Eigen::MatrixXd coriolis = model.compute_coriolis_matrix(js);
89 // coriolis torques, equivalent to coriolis multiplied by joint velocities. The joint positions part of the state
90 // is used to compute the coriolis matrix
91 state_representation::JointState js = state_representation::JointState::Random("myrobot", 7);
92 state_representation::JointTorques coriolis_t = model.compute_coriolis_torques(js);
93 
94 // gravity torques from joint positions
95 state_representation::JointPositions jp = state_representation::JointPositions::Random("myrobot", 7);
96 state_representation::JointTorques gravity_t = model.compute_gravity_torques(jp);
97 ```
